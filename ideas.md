## 方案一：Cybernetic Brutalism（赛博朴素主义）

**设计运动**：融合 Brutalism 的原始力量感与赛博朋克的科技未来感

**核心原则**：
1. 粗犷的排版对比 — 超大标题与精细正文形成极端对比
2. 暴露的结构 — 网格线、边框、节点连接线作为装饰元素可见
3. 高对比度色彩 — 深黑底色上的霓虹紫红渐变形成视觉冲击
4. 功能即美学 — 代码块、架构图本身就是设计的核心视觉元素

**色彩哲学**：以 #030712 深空黑为画布，#6D49FF 紫色代表"智能与创新"，#FF4646 红色代表"Rust 的力量与速度"，二者渐变象征从"AI 智能"到"Rust 性能"的无缝融合。

**布局范式**：打破传统居中对称，采用不对称网格布局。Hero 区域左文右图的经典分割，但通过倾斜的分割线和溢出边界的元素打破规整感。

**标志性元素**：
1. 旋转的 conic-gradient 边框 — 象征数据在网关中的流转
2. 粒子流动动画的连接线 — 展示请求从客户端到 LLM 的路径

**交互哲学**：每个交互都应该感觉像在操作一个精密的控制面板。Hover 时的发光效果模拟"激活"状态，滚动触发的渐显动画模拟"系统启动"。

**动画**：
- 入场动画模拟系统启动序列（依次点亮各模块）
- 架构图的连接线使用 stroke-dasharray 绘制动画
- 数字指标使用 easeOutCubic 缓动的计数器动画
- 卡片 hover 时的双层发光效果（紫色内光 + 红色外光）

**字体系统**：Sora（标题，几何感强）+ Outfit（正文，现代清晰）+ Fira Code（代码，等宽专业）

## 方案二：Liquid Topology（流体拓扑）

**设计运动**：受流体动力学和拓扑学启发的有机科技风

**核心原则**：
1. 流动的形态 — 所有边界和分割线都使用曲线和渐变过渡
2. 深度层叠 — 通过 blur、opacity 和 z-index 创造多层空间感
3. 有机的网格 — 非规则的卡片尺寸和间距，模拟自然生长
4. 呼吸感节奏 — 动画使用正弦波缓动，创造"活着"的感觉

**色彩哲学**：以深海蓝黑为基底，紫色渐变如同深海中的生物发光，红色点缀如同数据脉冲。整体营造"数据海洋"的意境。

**布局范式**：采用流体网格，卡片之间使用不等间距，大小交错排列。背景使用多层渐变叠加创造深度。

**标志性元素**：
1. 液态玻璃态卡片 — 高度模糊的背景 + 折射效果
2. 脉动的数据流线 — 使用 SVG 路径动画模拟数据在系统中流动

**交互哲学**：交互应该感觉像触碰水面 — 涟漪扩散、波纹传递。鼠标移动时的视差效果增强空间感。

**动画**：
- 背景使用缓慢流动的渐变动画
- 卡片入场使用弹性缓动（spring physics）
- 数据流线使用连续的路径动画
- 鼠标跟随的微妙视差效果

**字体系统**：Space Grotesk（标题，未来感）+ Inter（正文，中性清晰）+ JetBrains Mono（代码）

## 方案三：Industrial Precision（工业精密）

**设计运动**：受精密仪器和工业设计启发的极简科技风

**核心原则**：
1. 精确的网格系统 — 所有元素严格对齐到 8px 网格
2. 克制的装饰 — 仅使用线条、点和最少的色彩
3. 信息密度优先 — 在有限空间内传达最大信息量
4. 工程师审美 — 代码即设计，配置即界面

**色彩哲学**：几乎纯黑白的基调，仅在关键信息点使用品牌渐变色。这种克制让渐变色出现时更具冲击力，如同仪表盘上的警示灯。

**布局范式**：严格的模块化网格，每个区块都有明确的边界和标签。类似于技术文档的结构化布局，但通过精致的细节提升品质感。

**标志性元素**：
1. 细线边框和精确的圆角 — 模拟精密仪器的面板
2. 等宽字体的数据展示 — 数字和指标使用 monospace 字体

**交互哲学**：交互应该感觉像操作精密仪器 — 精确、可预测、有反馈。每个点击都有明确的状态变化。

**动画**：
- 极简的淡入动画，时长短促（200-300ms）
- 数字变化使用翻转动画（flip counter）
- 代码块使用打字机效果逐行显示
- hover 状态使用精确的 1px 边框变化

**字体系统**：IBM Plex Sans（标题，工业感）+ IBM Plex Sans（正文，统一性）+ IBM Plex Mono（代码）
